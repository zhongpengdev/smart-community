# 架构重构日志 (P0) - 2026-05-08

## 1. 重构主题：工程化与标准化基座升级
本次重构完成了《工业级上线就绪度审计报告》中定义的 **P0 级核心任务**，解决了困扰项目的环境硬编码与请求链路碎片化问题。

---

## 2. 环境配置解耦 (Environment Separation)
### 问题背景
- `nuxt.config.ts` 中硬编码了特定的 IP 地址 (`172.20.10.14`)。
- **风险**: 导致无法在不同环境（开发/测试/生产）间无缝切换，且存在内网拓扑泄露风险。

### 修复方案
- **引入环境变量**: 在根目录建立 `.env` 系统。
- **动态注入**: 重构 `nuxt.config.ts` 中的 `runtimeConfig`，使其在运行时自动映射环境变量。
- **统一调用**: 确保项目中所有基础设施地址均通过 `useRuntimeConfig()` 动态获取。

---

## 3. 请求链路标准化 (Request Layer Standardization)
### 问题背景
- 后台管理系统 (`superCommunity`) 大量滥用原生 `fetch`，绕过了全局 `$api` 网关。
- **后果**: 导致之前修复的“401 节流报错”在这些页面完全失效，且造成大量重复代码。

### 修复方案
- **建立契约层**: 在 `utils/api.ts` 中新增并封装了超过 30 个管理端契约函数。
- **全量模块重构**:
    - **用户管理**: 彻底拔除原生 `fetch`，统一鉴权逻辑。
    - **角色管理**: 重构复杂的权限分配逻辑，消除手动 Header 注入。
    - **权限资源**: 实现加载状态与错误处理的统一。
    - **物业费管理**: 修复残余的硬编码请求。
- **解耦 UI**: UI 组件现在不再感知 Token 注入和 URL 拼接细节。

---

## 4. 最终收益
1. **地基加固**: 全量业务现在均受全局网关（401 自动跳转、报错节流）保护。
2. **极简迁移**: 切换后端环境仅需修改 `.env.local` 并在 1 秒内重启，无需触碰源代码。
3. **维护性**: API 路径变更只需在契约层修改一次。
